# Systems as Attack Vectors

## Objective
Understand how attackers exploit vulnerable systems and services to gain unauthorized access, disrupt operations, or compromise organizational security.

---

## Concepts Covered
- System attack vectors
- Vulnerability exploitation
- Misconfigurations
- Unpatched software
- Weak authentication
- Exposed services

---

## System Attack Vectors

Attackers target systems by exploiting technical weaknesses within:
- Operating systems
- Applications
- Network services
- Authentication mechanisms

Common attack surfaces include:
- Public-facing services
- Remote access systems
- Web applications
- Open ports

---

## Common System Weaknesses

### Unpatched Software
Outdated systems may contain known vulnerabilities that attackers can exploit.

Risks include:
- Remote Code Execution (RCE)
- Privilege escalation
- Unauthorized access

---

### Misconfigurations
Improperly configured systems can expose sensitive services or weaken security controls.

Examples:
- Default credentials
- Open databases
- Excessive permissions
- Publicly exposed admin panels

---

### Weak Authentication
Weak passwords or poor authentication practices increase the risk of compromise.

Examples:
- Password reuse
- Simple passwords
- Lack of MFA

---

## Common Attack Techniques

### Malware
Malicious software designed to:
- Steal data
- Disrupt systems
- Gain persistence

Examples:
- Ransomware
- Trojans
- Spyware

---

### Brute Force Attacks
Attackers attempt multiple password combinations to gain access to accounts or services.

---

### Exploiting Vulnerabilities
Attackers use publicly known vulnerabilities to compromise systems and applications.

---

## Indicators of Compromise (IOCs)

Potential signs of system compromise:
- Unusual login activity
- Unexpected processes
- High network traffic
- Unauthorized file changes
- Suspicious outbound connections

---

## Defensive Best Practices

Recommended security measures:
- Regular patch management
- Strong authentication policies
- Multi-factor authentication (MFA)
- Principle of least privilege
- Continuous monitoring
- Secure system configuration
