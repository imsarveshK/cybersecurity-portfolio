# MITRE ATT&CK Framework

## Objective
Understand the fundamentals of the MITRE ATT&CK framework and learn how security teams use it to identify, analyze, and defend against adversary tactics and techniques.

---

## Concepts Covered
- MITRE ATT&CK
- Adversary tactics
- Attack techniques
- Threat intelligence
- Threat hunting
- Defensive security mapping

---

## MITRE ATT&CK Overview

MITRE ATT&CK is a globally recognized knowledge base that documents real-world adversary behaviors, tactics, and techniques observed during cyber attacks.

The framework helps organizations:
- Understand attacker behavior
- Improve threat detection
- Map security events to known techniques
- Strengthen defensive strategies

---

## Structure of the Framework

The MITRE ATT&CK framework is organized into:
- Tactics
- Techniques
- Sub-techniques

---

## Tactics

Tactics represent the attacker’s objectives during different stages of an attack.

Examples include:
- Initial Access
- Execution
- Persistence
- Privilege Escalation
- Defense Evasion
- Credential Access
- Discovery
- Lateral Movement
- Command and Control
- Exfiltration

---

## Techniques

Techniques describe how attackers achieve tactical objectives.

Examples:
- Phishing
- PowerShell abuse
- Credential dumping
- Remote service exploitation
- Scheduled task creation

---

## Sub-Techniques

Sub-techniques provide more detailed descriptions of specific attacker behaviors within a technique.

---

## Importance for SOC Analysts

SOC analysts use MITRE ATT&CK to:
- Investigate incidents
- Map attacker activity
- Improve threat hunting
- Develop detection rules
- Understand attack progression

The framework helps analysts identify suspicious behavior based on real-world adversary techniques.

---

## Threat Hunting & Detection

MITRE ATT&CK supports:
- Behavioral detection
- Threat intelligence analysis
- SIEM correlation
- Detection engineering

Analysts can build detections around known attacker techniques rather than relying only on basic IOCs.

---

## Defensive Strategies

Recommended approaches:
- Monitor attacker behaviors
- Map alerts to ATT&CK techniques
- Develop behavioral detections
- Harden systems against common attack methods
- Improve visibility across infrastructure
