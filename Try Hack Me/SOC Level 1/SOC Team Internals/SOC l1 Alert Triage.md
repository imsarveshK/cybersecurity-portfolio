# SOC L1 Alert Triage

## Objective
Learn the fundamentals of alert triage within a Security Operations Center (SOC), including identifying, analyzing, prioritizing, and responding to security alerts efficiently.

---

## Concepts Covered
- Alert triage
- Security event investigation
- False positives
- Alert prioritization
- Indicators of Compromise (IOCs)
- Incident escalation

---

## Alert Triage Overview

Alert triage is the process of:
- Reviewing security alerts
- Determining alert severity
- Validating suspicious activity
- Prioritizing incidents
- Escalating confirmed threats

The primary goal is to distinguish legitimate threats from benign activity efficiently.

---

## Alert Investigation Process

### Initial Review
Security analysts begin by:
- Reviewing alert details
- Identifying affected systems
- Checking timestamps and event sources
- Gathering contextual information

---

### Validation & Analysis
Analysts determine whether the alert is:
- A true positive
- A false positive
- A benign event

Investigation may involve:
- Log analysis
- Threat intelligence checks
- User activity review
- Network traffic analysis

---

## Alert Severity Levels

Common alert classifications:
- Low
- Medium
- High
- Critical

Severity depends on:
- Business impact
- Affected assets
- Threat confidence
- Scope of compromise

---

## Indicators of Compromise (IOCs)

Examples of IOCs used during triage:
- Malicious IP addresses
- Suspicious domains
- File hashes
- Unusual login attempts
- Unauthorized processes

---

## Common Challenges in Alert Triage

SOC analysts may encounter:
- Large alert volumes
- Alert fatigue
- False positives
- Incomplete data
- Time-sensitive investigations
