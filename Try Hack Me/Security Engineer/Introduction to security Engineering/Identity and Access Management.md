# Identity and Access Management (IAM)

## Objective
Understand the fundamentals of Identity and Access Management (IAM), including authentication, authorization, access control models, and security practices used to manage user identities and permissions securely.

---

## Concepts Covered
- Identity and Access Management (IAM)
- Authentication
- Authorization
- Access control
- Multi-Factor Authentication (MFA)
- Principle of Least Privilege
- Account security

---

## IAM Overview

Identity and Access Management (IAM) is the process of:
- Managing digital identities
- Controlling user access
- Enforcing authentication policies
- Protecting organizational resources

IAM helps organizations ensure that only authorized users can access systems and data.

---

## Authentication

Authentication verifies the identity of users attempting to access resources.

Common authentication methods include:
- Passwords
- Multi-Factor Authentication (MFA)
- Biometrics
- Security tokens

Strong authentication reduces unauthorized access risks.

---

## Authorization

Authorization determines what authenticated users are permitted to access or perform.

Examples:
- File permissions
- Administrative privileges
- Application access rights

Authorization policies help enforce security boundaries.

---

## Access Control Models

### Role-Based Access Control (RBAC)
Permissions are assigned based on user roles within an organization.

Examples:
- Administrator
- Analyst
- Standard User

---

### Principle of Least Privilege
Users should only receive the minimum permissions necessary to perform required tasks.

Benefits include:
- Reduced attack surface
- Limited privilege abuse
- Lower impact of compromise

---

## Multi-Factor Authentication (MFA)

MFA adds additional verification layers beyond passwords.

Examples:
- Authentication apps
- SMS verification
- Hardware tokens

MFA significantly improves account security.

---

## Account Security Risks

Potential IAM-related risks include:
- Weak passwords
- Credential reuse
- Excessive permissions
- Privilege escalation
- Compromised accounts

---

## IAM Best Practices

Recommended security measures:
- Enforce strong password policies
- Enable MFA
- Apply least privilege principles
- Review permissions regularly
- Monitor suspicious login activity
- Disable unused accounts

## Key Learnings
- IAM controls how users access systems and resources
- Authentication verifies identity while authorization controls permissions
- MFA improves account protection significantly
- Least privilege reduces organizational risk
- Proper IAM practices strengthen overall security posture
