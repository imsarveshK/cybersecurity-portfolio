# Phishing Analysis Tools

## Objective
Understand the tools and techniques used to analyze phishing emails, investigate suspicious indicators, and identify malicious content during phishing investigations.

---

## Concepts Covered
- Phishing analysis
- Email investigation tools
- URL analysis
- Attachment analysis
- Threat intelligence
- Indicators of Compromise (IOCs)

---

## Phishing Analysis Overview

Phishing analysis involves investigating suspicious emails to determine whether they contain:
- Malicious links
- Harmful attachments
- Spoofed sender information
- Social engineering techniques

SOC analysts use multiple tools to validate suspicious content and identify potential threats.

---

## Email Header Analysis Tools

Email header analysis helps analysts identify:
- Sender information
- Mail routing paths
- Spoofed addresses
- Suspicious domains

Common analysis tasks include:
- Reviewing Received headers
- Checking Reply-To addresses
- Identifying sender inconsistencies

---

## URL Analysis Tools

URL analysis tools help determine whether links are:
- Malicious
- Suspicious
- Legitimate

Analysts inspect:
- Domain reputation
- Redirect behavior
- URL structure
- HTTPS usage

---

## Attachment Analysis Tools

Attachments may contain:
- Malware
- Malicious macros
- Credential stealers
- Exploit payloads

Analysis methods include:
- Sandbox execution
- File hash checks
- Static analysis

---

## Threat Intelligence Platforms

Threat intelligence tools help analysts:
- Verify malicious IPs
- Check domain reputation
- Identify known malware hashes
- Correlate phishing indicators

## Investigation Workflow

Typical phishing investigation process:
- Review suspicious email
- Analyze email headers
- Inspect URLs
- Investigate attachments
- Validate indicators using threat intelligence
- Escalate confirmed phishing incidents

---

## Indicators of Phishing

Common phishing indicators:
- Suspicious domains
- Unusual sender addresses
- Fake login pages
- Malicious attachments
- Social engineering tactics

---

## Best Practices

Recommended approaches:
- Validate links before opening
- Analyze attachments in sandbox environments
- Verify sender authenticity
- Use threat intelligence during investigations
- Document phishing indicators clearly

---

## Key Learnings
- Multiple tools are required for effective phishing analysis
- Threat intelligence improves phishing investigations
- Email headers contain valuable investigation data
- Sandbox environments help analyze suspicious files safely
- Analysts must validate suspicious indicators carefully
