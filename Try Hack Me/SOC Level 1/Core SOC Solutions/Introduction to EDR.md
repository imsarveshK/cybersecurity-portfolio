# Introduction to EDR

## Objective
Understand the fundamentals of Endpoint Detection and Response (EDR), including how EDR solutions monitor endpoints, detect malicious activity, and support incident response operations.

---

## Concepts Covered
- Endpoint Detection and Response (EDR)
- Endpoint monitoring
- Threat detection
- Incident response
- Behavioral analysis
- Endpoint telemetry

---

## EDR Overview

Endpoint Detection and Response (EDR) solutions are security tools designed to:
- Monitor endpoint activity
- Detect suspicious behavior
- Investigate threats
- Support incident response efforts

EDR platforms provide visibility into activities occurring on systems such as:
- Workstations
- Servers
- Laptops

---

## Purpose of EDR

EDR solutions help security teams:
- Detect advanced threats
- Investigate suspicious activity
- Respond to incidents quickly
- Reduce attacker dwell time

EDR improves endpoint visibility by collecting detailed telemetry data.

---

## Endpoint Telemetry

EDR platforms collect data related to:
- Running processes
- File activity
- Network connections
- User logins
- Command execution

This telemetry helps analysts identify malicious behavior.

---

## Threat Detection

EDR tools use:
- Behavioral analysis
- Detection rules
- Threat intelligence
- Anomaly detection

to identify suspicious or malicious activity on endpoints.

---

## Common EDR Capabilities

Typical EDR features include:
- Process monitoring
- File integrity monitoring
- Threat hunting
- Alert generation
- Incident investigation
- Endpoint isolation

---

## Incident Response with EDR

SOC analysts use EDR platforms to:
- Investigate alerts
- Analyze endpoint activity
- Contain compromised systems
- Collect forensic evidence
